# 持久化源码导航

`Stores/` 保存配置与流程持久化的权威实现，并提供原子写入、联合事务和运行日志落盘能力。它不负责窗体交互、设备执行或流程调度。

| 目录 | 主要内容 | 职责 |
| --- | --- | --- |
| `Infrastructure/` | `AtomicJsonFileStore`、`ConfigurationBatchWriter` | JSON 原子读写与跨文件批量提交基础设施 |
| `Process/` | 流程 Repository、Work 事务、流程变量事务、流程日志 | 流程配置内存、工作目录事务、联合提交与运行日志落盘 |
| `Data/` | 报警、数据结构模型与 Store | 业务数据定义和报警配置持久化 |
| `Variables/` | 变量配置、变量调试配置 | 变量定义与调试映射持久化 |
| `Devices/` | 控制卡、通讯、PLC、IO、工站和托盘点位 | 设备及通讯配置持久化，不执行设备动作 |

新增持久化文件应进入对应事实域；`Stores/` 根目录只保留本导航文档。
