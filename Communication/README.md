# 通讯运行时源码导航

`Communication/` 是 TCP 与串口通讯枢纽。`CommunicationHub.cs` 包含通道、共享监听、会话路由、原始数据接收分发和事务等待；`CommunicationModels.cs` 保存跨层使用的通讯配置模型。

## TCP 当前契约

- `LocalAddress`、`LocalPort` 始终表示本机绑定端点；`RemoteAddress`、`RemotePort` 始终表示对端端点或服务端允许接入的对端条件，不再根据 Client/Server 改变字段含义。当前地址契约只接受 IPv4。
- Client 必须配置远端 IP 和非零远端端口；本地端口允许为 `0`，表示由系统分配源端口。连接前始终显式绑定本地端点。
- Client 的 `AutoReconnect=true` 同时覆盖首次连接失败和已连接后断线，退避为 1、2、5、10、30 秒，停止通道会取消重连。
- Server 必须配置非零本地监听端口。远端 IP 可为具体 IPv4 或 `*`，远端端口可为 `0`；`* : 0` 是该监听端点唯一的兜底通道。
- 多个 Server 逻辑通道可共享同一实际监听端点。连接按“具体 IP+端口、具体 IP、通配条件”的优先级分配；具体远端通道只占用一个会话，发送和接收均隔离到该会话。通配通道可接纳多个客户端，发送保持广播语义。
- `TcpStatus` 分开报告 `IsStarted`、`IsConnected` 和 `ConnectionState`。监听中、连接中、重连中不伪装成已连接，流程等待仍以真实连接状态为准。
- TCP 与串口均直接交付底层每次读取到的原始字节，不按分隔符拆分，文本发送也不自动追加结束符。协议需要分包时由调用方基于收到的数据处理。

配置由 `CommunicationConfigStore` 校验并原子保存；运行时只消费已验证快照。监听地址重叠、客户端固定本地端点冲突、重复远端选择条件以及同一监听端点的多个兜底通道都会在保存阶段失败。

## 流程业务重试

连接生命周期的 `AutoReconnect` 仍由 `CommunicationHub` 管理；流程业务请求的重试由 Engine 中的通信指令管理，两者不是同一个机制。TCP/串口数据通信指令只提供额外重试次数和固定重试间隔，不使用业务退避。接收结果条件只在重试次数大于0时执行；次数为0时只通信一次并跳过判定。发送请求可能在远端已经生效但本地未收到确认，因此启用发送重试前必须确认协议允许重复下发。

当前只有两个高度内聚文件，保持扁平结构。只有出现新的独立协议实现时才增加子目录，不为现有类型增加包装层。
