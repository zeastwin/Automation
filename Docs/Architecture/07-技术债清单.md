# 技术债清单

本文只记录有源码证据、实际风险和验收条件的问题。完成过程由 Git 保存，不在这里长期堆积逐批改造记录。

优先级含义：

- `P1`：继续扩展会放大一致性、安全或契约风险，应优先处理；
- `P2`：影响可读性、可测试性或维护成本，在相关模块再次扩展前处理；
- `P3`：机械清理，并入相邻改动，不单独制造大批次。

## 架构重整停止结论

截至 2026-07-22，没有需要继续发起大范围架构改造的 `P1/P2` 项。当前停止以“继续拆文件、继续抽接口、继续增加程序集”为目标的重整，转入正常产品维护。

只有以下事实允许重新启动架构工作包：

- 同一事实再次出现两个可写入口，或出现绕过事务、安全闸门、readiness 的路径；
- 一个模块的变化持续导致三个以上无关模块联动，且现有自动测试无法隔离定位；
- 实测构建、部署、性能、复用或框架生命周期问题无法在当前 `.NET Framework 4.8 + WinForms` 边界内解决。

单纯行数较多、目录还能再细分或理论上可以再抽一个接口，不构成重新开工的理由。

## 必须保持的已完成边界

| 边界 | 当前结果 | 防回流证据 |
| --- | --- | --- |
| 实例所有权 | `PlatformRuntime`、具体 Store 和 `EditorWorkspace` 替代 `SF` 与静态窗体引用。 | 架构门禁扫描 `SF.*`、旧全局入口和隐藏编辑器依赖。 |
| UI 与内核 | Engine、Store、MotionControl 不依赖 WinForms；弹窗和刷新通过契约与适配器完成。 | 依赖扫描、无 UI Engine 测试和窗体构造测试。 |
| 配置所有权 | Store 持有配置事实；流程与变量通过联合事务提交；安全停止进入统一协调器。 | 事务/恢复测试以及 UI、Bridge 直接文件写入门禁。 |
| 流程编辑 | 稳定 ID、共享选择状态、指令编辑服务和流程变量提交服务构成正式编辑边界。 | 编辑、跳转、回滚和流程图回归。 |
| AI 写入 | ChangeSet V2 `actions -> preview -> confirm -> apply(previewId)` 是唯一流程结构写入链。 | 旧路由/字段门禁、Profile/Schema 校验和状态机测试。 |
| 生命周期 | 编辑器与设备宿主共用幂等关闭链；重复启动保留当前实例并返回错误。 | Engine 生命周期和关闭协调测试。 |
| 物理结构 | 应用、Editor、Runtime、Engine、Stores、Bridge、MotionControl 按最小职责归档；无 UI 契约独立成程序集。 | 项目项、目录 README、程序集依赖和 Debug/Release 构建。 |
| 自动验证 | 自动回归统一进入 `Automation.Core.Tests`，旧散落脚本由同一测试项目调度。 | `dotnet test Automation.sln -p:Platform=x64` 和编译前架构门禁。 |

## 记录新债务的门槛

新增条目必须同时写清：

- 源码或运行日志证据；
- 实际风险和受影响范围；
- 当前权威事实源；
- 需要删除的重复事实、旁路或旧实现；
- 可自动或人工验证的退出条件。

只有“代码很乱”而没有上述证据的问题，不进入清单。
