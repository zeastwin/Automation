# 技术债清单

## 使用方式

本文只记录有源码证据、能定义验收条件的债务。优先级含义：

- `P1`：下一轮应优先收敛，继续扩展会放大数据一致性、安全或契约风险。
- `P2`：重要的可读性和可测试性问题，应在相关模块再次扩展前处理。
- `P3`：机械性清理，适合并入相邻改造，不单独制造大变更。

截至 2026-07-22 本轮收敛后的规模基线：`AutomationBridgeService.cs` 主文件约 125 行；`FrmAiAssistant.cs` 主文件约 2,475 行；`FrmIODebug.cs` 约 2,820 行；`FrmValue.cs` 约 2,780 行；`FrmProc.cs` 约 2,130 行；`FrmDataGrid.cs` 约 1,920 行；`InspectorView.cs` 约 485 行；`FrmMain.cs` 约 955 行。数字只用于确定拆分优先级，不作为代码质量结论。
